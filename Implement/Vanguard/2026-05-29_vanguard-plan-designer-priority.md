# Project Ember Vanguard - 기획 작업 우선순위 (2026-05-29)

> 상위 문서: [[2026-05-29_vanguard-implementation-plan-overview]]
> 짝 문서: [[2026-05-29_vanguard-implementation-plan-outgame]] / [[2026-05-29_vanguard-implementation-plan-ingame]]

기획자가 어떤 순서로 스펙/데이터를 채워야 개발이 막히지 않는지 정리한 문서.
각 단계에 **기획자가 바로 채울 수 있는 빈 템플릿**을 포함한다.

---

## 작업 원칙

```
✅ 구조 정의(위키 기반, 확정 가능) → 먼저, 빨리 확정 (개발 언블록 + 잘 안 바뀜)
✅ 밸런싱 수치(플레이테스트 필요) → 임시값 먼저 주고, 나중에 다듬기
   ※ 밸런싱을 완벽히 정한 뒤 개발 시작 ❌ → 개발이 멈춤. 임시값으로 흐름 먼저.
✅ 기획은 항상 개발 마일스톤보다 한 발 앞서 산출물을 낸다.
```

확정 난이도 표기:
- 🟢 **지금 확정**: 위키/구조 기반, 변경 거의 없음
- 🟡 **임시값 먼저**: 일단 placeholder, 플레이테스트 후 확정
- 🔴 **후순위**: 전투/점수 확정 후에야 의미 있음

## 기획자가 놓치기 쉬운 3가지

1. **③ 칩 인벤토리 정책을 P0에서 못 박을 것.** "Vanguard 칩 = 기존 영구칩과 분리 + 매주 리셋"이 PvP 공정성의 핵심. 늦게 정하면 개발이 영구 저장 구조로 만들어 재작업.
2. **⑦ 점수식은 완벽히 정하려 하지 말 것.** 임시 가중치 먼저 → 전투 돌려보고 다듬기.
3. **⑱ 봇 클론 더미 빌드는 기획 산출물.** 시즌 첫날 클론 풀이 비면 매칭 불가(위키 "초기 매칭 문제"). 개발이 아닌 기획이 미리 설계.
